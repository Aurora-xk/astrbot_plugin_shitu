## 🎯 功能特色

✨ **智能图片识别**: 基于 [AnimeTrace](https://ai.animedb.cn/)，自动识别动漫、GalGame、二次元游戏角色  
💡 **动态模型列表**: 启动时自动从 API 获取当前可用模型，无需手动更新  
🛡️ **完善的错误处理**: 30 秒超时保护，异常捕获，优雅降级  
📝 **引用消息支持**: 可以识别引用消息中的图片  
🎨 **美观的结果展示**: 清晰的格式化输出，支持多结果显示  
🖼️ **角色图片返回**: 可选按识别框裁剪角色区域，并发送多张角色图片  
🤖 **智能头像识别**: 支持 @用户、手动输入 QQ 号、自动识别自己头像  
📤 **QQ 合并转发**: 识别到多个角色时自动以聊天记录形式合并转发（仅 aiocqhttp）  
⚙️ **可配置化**: 支持自定义超时时间、提示文字、结果数量、合并转发阈值等

## 📋 识别命令

| 🎌 命令 | 🎯 功能说明 |
|--------|------------|
| `/识别` | 识别图片中的二次元角色 |
| `/头像识别` | 识别 QQ 头像 |
| `/amt model` | 查看当前可用模型列表 |

## 🚀 使用方式

### 方式1: 直接识别（推荐）
图片和指令一起发送：
```
[图片] + /识别
```

### 方式2: 等待模式
先发送识别命令，再发送图片：
```
/识别
📷 请发送要识别的图片（30秒内有效）
[图片]
```

### 方式3: 引用识别
引用包含图片的消息并发送识别命令：
```
[引用消息包含图片] + /识别
```

### 方式4: QQ 头像识别
支持多种智能方式识别 QQ 头像：

#### 🎯 @用户识别
```
@目标用户 /头像识别
```

#### 🆕 手动输入 QQ 号识别
```
/头像识别 123456789
```

#### 🤖 智能自我识别
```
/头像识别
```

💡 **智能行为**: 
- **@目标用户**: 识别指定用户的头像
- **手动输入 QQ 号**: 识别指定 QQ 号的头像
- **不@任何人**: 自动识别发送者自己的头像

⚠️ **注意**: 
- **正确示例**: `@小明 /头像识别` （识别小明的头像）
- **手动示例**: `/头像识别 123456789` （识别 QQ 号 123456789 的头像）
- **智能示例**: `/头像识别` （识别自己的头像）




## ⚙️ 配置说明

插件支持通过 `_conf_schema.json` 文件进行配置，主要配置项包括：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `timeout_seconds` | 等待用户发送图片的超时时间 | 30 |
| `prompt_send_image` | 提示用户发送图片的文字 | "📷 请发送要识别的图片（30秒内有效）" |
| `prompt_timeout` | 用户超时后的提示文字 | "⏰ 识别请求已超时，请重新发送命令" |
| `return_crops` | 是否在文字结果后发送裁剪出的角色图片 | true |
| `max_crops` | 每次识别最多发送的裁剪图片数量 | 5 |
| `max_characters_per_role` | 每个角色最多显示几个匹配结果 | 5 |
| `forward_threshold` | 识别到多少个角色时自动使用 QQ 合并转发 | 不开启 |

## 📋 更新日志

### 📅 2026 年 7 月 4 日 - v4.0
#### ✨ 新增功能
- 对接新版 AnimeTrace API，自动从 `https://api.animetrace.com/v1/model/list` 获取可用模型
- 新增 `/amt model` 和 `/amt model N` 命令，支持查看和切换识别模型
- 新增 `forward_threshold` 配置项，识别到多个角色时自动使用 QQ 合并转发
- 新增 `max_characters_per_role` 配置项，可自定义每个角色显示的结果数量

#### 🔧 优化改进
- 移除已弃用的 Gal 识别、动漫识别分类命令，统一为 `/识别`
- 优化错误码处理，根据 API 返回的 code 给出更准确的提示

#### �️ 移除功能
- 删除 `/动漫识别`、`/gal识别`、`/头像动漫识别`、`/头像gal识别` 命令
- 删除硬编码模型映射，改为动态获取

### � 2025 年 12 月 2 日 - v3.6.0
#### � Bug 修复
- 修复 AnimeTrace 返回多个角色框时，仅展示第一组角色的异常，现在会按每个角色单独分组展示结果。

#### ✨ 新增功能
- 新增可选的「发送裁剪角色图片」功能，可根据识别结果的 box 裁剪人物区域并发送图片。
- 新增配置项 `return_crops`（是否发送裁剪图片）与 `max_crops`（每次最多发送的裁剪图片数量）。
